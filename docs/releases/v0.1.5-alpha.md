# SilkMC v0.1.5-alpha

SilkMC v0.1.5-alpha expands the server's compatibility tooling, operator visibility, and CI coverage for Paper, Bukkit, and Spigot plugins on a region-threaded runtime.

## Highlights

- Added per-plugin compatibility overrides through `silkmc-plugin-overrides.yml`.
- Added the `/silkmc` operator command with status, plugin listing, report, and reload subcommands.
- Added a curated plugin compatibility smoke test workflow for pull requests and pushes to `main`.

## Compatibility changes

- Operators can override plugin classification to `SAFE`, `COMPATIBLE`, `UNSAFE`, `UNKNOWN`, or `AUTO`.
- Force-load overrides can bypass strict-mode for reviewed plugins that would otherwise stay blocked.
- Override usage is logged at `INFO` level with the configured operator reason for auditability.

## Operational notes

- `/silkmc reload` refreshes `silkmc-compatibility.yml` and `silkmc-plugin-overrides.yml` without rescanning plugin jars.
- The CI smoke test currently boots SilkMC with LuckPerms, EssentialsX, Vault, WorldEdit, and ProtocolLib from pinned official sources.

## Known limitations

- SilkMC still prefers blocking over unsafe world access when a plugin assumes a single global main thread.
- Reloading compatibility config does not invalidate cached inspection reports from the current process lifetime.
